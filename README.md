TESTE DO BANCO DE DADOS IMPLEMENTADO NO INTELLIJ IDEA
Requisitos Funcionais

RF001 - O sistema deve permitir conexão com o banco de dados MySQL utilizando JDBC.

RF002 - O sistema deve permitir cadastrar produtos na tabela produto.

RF003 - O sistema deve armazenar o nome do produto.

RF004 - O sistema deve armazenar a quantidade em estoque do produto.

RF005 - O sistema deve armazenar a quantidade mínima do produto.

RF006 - O sistema deve armazenar a data de validade do produto.

RF007 - O sistema deve permitir consultar produtos cadastrados.

RF008 - O sistema deve permitir atualizar informações dos produtos.

RF009 - O sistema deve permitir excluir produtos cadastrados.

RF010 - O sistema deve executar operações SQL de INSERT, SELECT, UPDATE e DELETE.

RF011 - O sistema deve permitir integração contínua utilizando GitHub Actions.

RF012 - O sistema deve conectar ao banco remoto durante os testes da CI.

Requisitos Não Funcionais

RNF001 - O sistema deve ser desenvolvido em Java.

RNF002 - O banco de dados utilizado deve ser o MySQL.

RNF003 - A conexão com o banco deve utilizar JDBC.

RNF004 - O sistema deve funcionar em ambiente Windows e Linux.

RNF005 - O código-fonte deve ser armazenado no GitHub.

RNF006 - O sistema deve utilizar GitHub Actions para integração contínua.

RNF007 - As credenciais do banco devem ser armazenadas utilizando GitHub Secrets.

RNF008 - O sistema deve apresentar mensagens de erro em caso de falha na conexão.

RNF009 - O workflow da CI deve executar automaticamente a cada push na branch principal.

RNF010 - O projeto deve possuir organização adequada de arquivos e pastas.
